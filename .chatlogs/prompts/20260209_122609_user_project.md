Functional Requirements
3.1. Data Aggregation
Automatic Parsing: The application must collect information about events from official websites of Wrocław theaters (e.g., Opera House, Polish Theatre, Contemporary Theatre, etc.).
Data Updates: Data should be updated at least once a day.
Error Handling: The system must notify administrators of failed parsing attempts or missing data on source websites.
3.2. Search and Filtering
Keyword Search: Users can search by event name, actor name, director, or theater name.
Category Filters:
Type of performance (ballet, opera, play, comedy, drama, musical, concert, etc.).
Date and time (today, tomorrow, this week, this weekend, specific date).
Venue (specific theater or city district).
Price range (free, up to 50 PLN, 50–100 PLN, 100–200 PLN, premium).
Age restrictions (0+, 6+, 12+, 16+, 18+).
Additional Filters:
Language of the performance (Polish, English, Russian, no words, etc.).
Duration (under 1 hour, 1–2 hours, over 2 hours).
3.3. Ranking and Sorting
Sorting Options:
By date (nearest to farthest).
By popularity (based on views or user saves).
By rating (if a rating system is implemented).
By price (ascending or descending).
Manual Ranking: Administrators can highlight specific events (e.g., "Recommended" or "Popular").
3.4. Event Card
Detailed Information:
Name, type of performance, brief and full description.
Date, time, and duration.
Venue (address, theater contact, map/directions).
Cast and director.
Ticket prices and purchase link (if available).
Age restrictions.
Poster or image gallery.
Interactive Elements:
"Buy Ticket" button (link to the theater’s official website or booking system).
"Add to Favorites" button.
"Share" button for social media.
3.5. User Account
Registration/Login: Via email, Google, or Facebook.
Favorites: Users can save interesting events to their account.
Notifications:
Customizable alerts for new events in selected categories.
Reminders for upcoming events from favorites.
Viewing History: List of previously viewed events.
3.6. Notifications
Email Newsletters: Weekly digest of events tailored to user interests.
Push Notifications: Reminders about events from favorites one day before they start.
3.7. Admin Panel
Content Management:
Manual addition, editing, or deletion of events.
Moderation of reviews and ratings (if implemented).
Statistics:
Popular events and user activity.
Parsing errors and update logs.
3.8. Additional Features (Optional)
Reviews and Ratings: Users can leave reviews and rate events.
Interactive Map: Display of all Wrocław theaters with current event markers.
Calendar Integration: Option to export events to personal calendars (Google Calendar, iCal).
